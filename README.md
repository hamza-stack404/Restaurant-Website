# Ember & Salt — Restaurant Website

A single-page restaurant website for **Ember & Salt**, a wood-fired coastal kitchen. Built with **HTML, Tailwind CSS (CDN), and vanilla JavaScript** — no frameworks, no build step required.

## Live Sections

- **Home / Hero** — Full-screen landing section with animated rising ember particles, a fade-in entrance sequence for the headline and text, and an animated scroll indicator.
- **Menu** — Four categories (Fire, Sea, Garden, Sweet & Bar), each dish shown with a photo, name, description, and price using a dotted price-leader style.
- **About** — Restaurant story, key stats, and a feature image.
- **Gallery** — Responsive image grid with hover zoom effects.
- **Reservations** — A client-side reservation form (date, time, guests, contact info, special requests) with instant confirmation message. No backend required.
- **Contact / Footer** — Address, contact details, hours, and social links.

## Tech Stack

| Technology | Purpose |
|---|---|
| HTML5 | Page structure and content |
| Tailwind CSS (via CDN) | Layout, spacing, color palette, typography, responsiveness |
| Custom CSS (`<style>` block) | Animations, keyframes, and effects Tailwind cannot express |
| Vanilla JavaScript | Interactivity (nav toggle, ember particles, form handling) |
| Google Fonts | Fraunces (display), Inter (body), IBM Plex Mono (mono/labels) |

## Animations & Effects

This version of the project **includes all animations**:

- **Ember particles** — 28 glowing dots are generated by JavaScript and animated rising upward across the hero section using a custom `@keyframes rise` animation.
- **Fade-up entrance** — The hero headline, subtext, and meta info fade and slide into view on page load (`@keyframes fadeUp`), staggered with delays.
- **Animated scroll indicator** — A line at the bottom of the hero pulses downward continuously (`@keyframes slideDown`).
- **Nav link underline** — Navigation links get an animated underline on hover.
- **Gallery hover zoom** — Images scale and saturate smoothly on hover.
- **Reduced motion support** — All animations respect the `prefers-reduced-motion` accessibility setting and are disabled for users who request it.

> A separate, animation-free version of this project also exists for environments where custom CSS/animations are restricted (e.g. assignment constraints requiring Tailwind-only styling).

## Why Some Custom CSS Is Still Used

While the layout, colors, spacing, and typography are handled entirely by Tailwind utility classes, a small `<style>` block is required for things Tailwind cannot do out of the box:

- Custom `@keyframes` animations (ember particles, fade-ups, scroll indicator)
- Pseudo-element styling (`::after` for the nav underline and scroll line)
- Styling the native `<select>` dropdown arrow icon
- The `prefers-reduced-motion` accessibility media query

This is standard practice even in production Tailwind projects.

## Getting Started

No installation needed — this is a static site.

1. Download or clone the project.
2. Open `index.html` directly in any modern browser.

To deploy:

- **GitHub Pages / Vercel / Netlify** — push the repository and deploy as a static site. No build command is required.

## Customization Notes

- **Images**: Menu, gallery, and about-section images currently use placeholder URLs (`picsum.photos`). Replace these with real photos of your dishes and restaurant before publishing.
- **Menu items & prices**: Edit directly inside the `#menu` section in `index.html`.
- **Colors & fonts**: Defined in the `tailwind.config` script block at the top of `index.html` (`charcoal`, `ember`, `gold`, `bone`, etc.).
- **Reservation form**: Currently client-side only (shows a confirmation message, no data is sent anywhere). Connect it to a backend or form service (e.g. Formspree, Netlify Forms) to receive real bookings.

## Production Note

This project uses the **Tailwind CSS CDN build** for simplicity. The browser console may show a warning recommending the Tailwind CLI/PostCSS setup for production — this is a performance recommendation only and does not affect functionality or deployment.
